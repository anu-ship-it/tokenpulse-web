ChatGPT doesn't tell you how much of the context window you've used. There's no indicator, no warning before the model starts forgetting the beginning of your conversation. You find out when your responses get worse, or when the model starts contradicting things it said earlier.

For developers using ChatGPT for long coding sessions, this is a real workflow problem.

## What Is the ChatGPT Context Window?

The context window is the maximum amount of text — your messages plus ChatGPT's responses — that the model can hold in memory at once. When a conversation exceeds the context window, older messages get dropped. The model can no longer reference them.

GPT-4o has a 128k token context window. That sounds large, but a complex multi-file codebase with detailed explanations can fill it in under an hour of active use.

## Why ChatGPT Doesn't Show Token Usage

OpenAI built ChatGPT as a consumer product. Consumer products hide complexity. Token counts are a technical concept most users don't need — so they're not surfaced in the UI.

For developers, this is a UX gap that costs real time when sessions get cut short unexpectedly.

## How a Token Counter Extension Works

A Chrome extension can read the DOM of the ChatGPT interface to estimate how many tokens have been used in the current conversation. It does this by counting characters in the conversation thread and applying the standard approximation of 1 token ≈ 4 characters for English text.

This isn't exact — tokenization varies by language and content type — but it's accurate enough to give meaningful early warning. An 8% margin of error on a 128k token window is still 10,000 tokens of headroom.

## What to Look for in a Token Counter Extension

**Accuracy**: Character-based estimation is good enough. Exact token counts require calling OpenAI's tokenizer on every message, which adds latency and requires API access.

**In-page display**: A popup you have to click to open is less useful than a bar that's always visible. You shouldn't have to check your usage — it should be impossible to miss.

**Notifications**: An alert at 75% and 90% is more valuable than one at 100%. At 100% it's too late.

**Multi-platform**: If you use Claude and ChatGPT simultaneously (most developers do), a tool that works on both is better than two separate extensions.

## What You Can't Track in ChatGPT

Unlike Claude, ChatGPT doesn't expose rate limit data through a session-accessible API. You can track context window usage, but you can't see hard message-per-period limits the same way.

This is a ChatGPT API limitation, not an extension limitation. Any extension claiming real-time ChatGPT rate limit data beyond the context window is either approximating or making it up.

## Tips for Managing ChatGPT Context

**Split tasks into separate chats.** Research in one chat, writing in another. Don't carry the full research thread into the output session.

**Use Custom Instructions for stable context.** Anything you repeat every session (your role, your tone preferences, your tech stack) belongs in Custom Instructions, not in the conversation.

**Summarize before long sessions.** Before a complex debugging session, summarize the problem statement and constraints in the first message. This front-loads the important context before any tokens are wasted.

**Watch the bar at 70%.** At 70% context usage, start thinking about how to wrap up or what to carry into a new session. At 90% you're already in degraded territory.
